My approach to build the assistant

From fantacy I thought about a website that has a inbuilt Voice Assistant to do every single operation of the website. So I decided to build it in My Portfolio website.

I started to study google assistant and cortana to get a besic idea about the basic functions to build my own, I started to learn about building voice assistant from YouTube and started to build my own voice assistant

I started to build it part by part and finally intigrated all the parts together

PRE-DEFINED functions

As I have seen in the case of Google Assistant and Cortana that there are some basic feedbacks that are predefined and the assistant repeats them randomly from provided arry of strings. So I predefined some of the answers for specific questions.

Using keywords instead of predefined statements

I made it a keywords based approach to increase the freedom to command as in this case "wh" words with some predefined words will lead to a specific reasult with lesser failure chance.

Making of the Navigation through voice

The voice assistant would be incomplete with out the power to execute the user commands. One of the most useful features is able to navigate through voice only.

It was complicated to find out the desired destination of the visitor, after some trial and errors I decided to go with the easiest possibility that I could think of. So, once again I used the keywords to split the string just after the word "To" and That is how the assistant knows visitors desired destination.

To go to the destination I used the HTML ID attribute to find the location and used click functions

Weather reaport

To get the real time weather reaport of any city I had to use "OpenWeathermap" API to get the weather reaport. I used the same strategy of keywords to get the city from the string.
